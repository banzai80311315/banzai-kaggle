# Power Forecast Project

Kaggle の [Hourly Energy Consumption Dataset](https://www.kaggle.com/datasets/robikscube/hourly-energy-consumption) を題材として、電験3種で学ぶ電力需給の知識とデータサイエンスを融合させて学習します。  
目標は「気温・曜日・時間帯などから電力需要を予測するモデルを構築」し、電力業界の需給計画・負荷平準化への理解を深めることです。

## 学習計画

| ステップ | 内容 | 目的 |
|:--|:--|:--|
| **① Kaggleの問題理解** | コンペページを読み、何を予測する課題か把握 | 分析のゴールを明確にする |
| **② データ構造の確認** | CSVを読み、カラム名・データ期間・地域などを確認 | どんな特徴量が作れそうか考える |
| **③ 可視化** | 時系列グラフ・季節性・曜日パターンなどを描画 | 電験的な「負荷特性」を直感的に理解 |
| **④ 前処理** | 欠損処理、日付分解（hour, weekday, monthなど） | 学習データを整える |
| **⑤ モデル構築（ベースライン）** | 線形回帰でまず1つ予測モデルを作る | 精度指標を確認して方向性を掴む |
| **⑥ 改良・考察** | ARIMA・XGBoost・LSTM などを試す | 時系列予測の理解を深める |
| **⑦ 電験的考察** | 気温・季節要因や負荷平準化の視点で結果を解釈 | 電力業界面接で話せる知見を得る |

## 電気主任技術者試験との関連

| 区分 | 電験での学び（理論・目的） | Kaggle分析での対応 | 学べるデータ分析スキル |
|:--|:--|:--|:--|
| **発電** | 各発電方式（火力・水力・原子力・再エネ）の特性を理解し、需給バランスを保つ | **電力需要予測（AEPデータ）**を用いて、どれだけ発電すべきかをモデルで予測 | 時系列予測（ARIMA, LSTM）、特徴量設計、評価指標（RMSE） |
| **出力制御** | 発電機の出力調整・負荷追従・周波数維持 | 予測誤差を最小化するモデル改善（ピーク検知・リグレッション） | モデル最適化、残差分析、制御的思考 |
| **送電** | 高電圧で効率的に送る（損失低減、潮流制御） | 需要データから送電負荷ピークを分析し、送電ロス・潮流分布をシミュレーション | 数値解析・線形代数・最適化モデリング |
| **送電損失** | 抵抗損（I²R）を減らす工夫（電圧昇圧・力率改善） | 電力データ×電圧・電流の仮想データを組み合わせて、損失を定量化 | Python数値計算、パラメトリックシミュレーション |
| **系統運用** | 周波数・電圧安定度、負荷追従、事故時復旧 | 予測誤差の動的評価、異常検知モデル（AI制御の入口） | 統計的プロセス制御（SPC）、異常検知（IsolationForest等） |
| **配電** | 地域ごとの負荷特性、配電損失、保護協調 | Kaggleデータの地域別CSV（AEP, COMED, PJM_Load）を比較して特性を抽出 | クラスタリング・EDA・可視化（matplotlib, seaborn） |
| **負荷特性** | 昼夜・季節・曜日の変化を理解する | 時系列データを分解し、季節性・トレンド・残差を分析 | 時系列分解（seasonal_decompose）、周期分析（FFT） |
| **力率改善** | 無効電力を抑え、皮相電力を減らす | ピークシフトや需要平準化（Demand Response）のモデル化 | 平滑化・移動平均・需要曲線解析 |
| **系統全体の安定化** | 発電から需要までの最適制御 | 各地域データの予測を統合し、PJM全体（PJM_Load）を再現 | マルチ系列時系列解析、アンサンブル学習 |
| **再生可能エネルギーの調整** | 出力変動を吸収して安定運用 | 太陽光・風力発電データ（別Kaggle）と組み合わせた需給予測 | マルチモーダル時系列、再エネ統合モデル |
